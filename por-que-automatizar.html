Por qué la integración entre SAP y Tookane no es óptima

La conexión existe, pero tiene cuatro problemas concretos que generan trabajo manual y pérdida de visibilidad:

1. Los datos llegan, pero sin estructura aprovechable La mayoría de los campos de SAP llegan a Tookane dentro de un contenedor genérico llamado extraInfo. Tookane los recibe como texto libre y no puede usarlos para automatizar decisiones: asignar carrier, gestionar plazos o disparar alertas según el tipo de envío.

2. Los datos de pedido no llegan a la capa logística De las 4.009 expediciones registradas, ninguna tiene número de pedido SAP, código de cliente ni importe vinculado. SAP gestiona los pedidos en la tabla VBAK y esa información nunca se transfiere a Tookane ni a la base logística. Si hay una incidencia, hay que buscar el pedido a mano en el ERP.

3. Sincronización manual y tardía El estado de cada entrega se actualiza cuando alguien descarga el CSV de Tookane manualmente, normalmente 1-2 veces por semana. Los sistemas internos siempre van con varios días de retraso respecto a la realidad del transporte.

4. Expediciones sin trazabilidad cruzada 584 expediciones existen en Tookane pero no tienen enlace verificado a SAP. No hay campo común que vincule de forma fiable el pedido SAP con su expedición en Tookane, lo que hace imposible la reconciliación automática.

SAP ↔ N8N ↔ Tookane

Por qué N8N es el intermediario, no solo un cable de conexión

SAP (sistema de gestión de Actiu) y Tookane (plataforma de gestión de transportes) no hablan el mismo idioma. Sin un intermediario que traduzca, los datos llegan incompletos o en un cajón genérico que Tookane no puede procesar automáticamente. N8N es ese intermediario.

1La información del envío está repartida en varios módulos de SAP
⚠ Sin N8N — SAP directo a Tookane

SAP (el sistema de gestión de Actiu) guarda los datos de cada envío en módulos separados: uno para el transporte (fechas, ruta, transportista), otro para el pedido de venta (referencia, cliente, importe), otro para la dirección de entrega, y otro para los artículos y bultos. Una conexión directa entre SAP y Tookane solo puede consultar un módulo a la vez y no puede ensamblar todos esos datos en un único envío.

✓ Con N8N como intermediario

N8N actúa como un asistente que recorre todos los módulos de SAP, recoge la información de cada uno y la reúne en un paquete completo. Tookane recibe todos los datos del envío de una sola vez, sin tener que ir a buscarlos en varios sitios.

2SAP y Tookane identifican a los transportistas de forma diferente
⚠ Sin N8N — SAP directo a Tookane

SAP identifica a cada transportista con un código de proveedor interno (un número como «1000045» que internamente significa Dachser). Tookane usa sus propios identificadores. Sin un traductor en el medio, Tookane recibe ese código numérico y no sabe a qué carrier corresponde.

✓ Con N8N como intermediario

N8N tiene en su base de datos la equivalencia entre el código interno de SAP y el nombre del transportista en Tookane. Antes de enviar, traduce automáticamente el código al identificador correcto para que Tookane asigne el carrier sin ambigüedad.

3SAP y Tookane escriben las fechas de manera distinta
⚠ Sin N8N — SAP directo a Tookane

SAP almacena las fechas en un formato propio: el 15 de junio de 2026 se guarda como «20260615», sin separadores ni hora. Tookane espera las fechas en el formato internacional estándar («2026-06-15»). Son el mismo dato pero escritos de forma incompatible.

✓ Con N8N como intermediario

N8N convierte automáticamente cada fecha del formato SAP al formato estándar antes de enviarla a Tookane. El proceso es invisible pero necesario para que las fechas de carga y entrega lleguen correctas y Tookane pueda programar rutas y alertas sin errores.

4SAP crea el envío antes de que esté listo para notificar a Tookane
⚠ Sin N8N — SAP directo a Tookane

En SAP, el documento de transporte se crea desde la primera fase de planificación, cuando todavía pueden faltar datos clave: el transportista no está asignado, la fecha de entrega no está confirmada. Si se enviara a Tookane en ese momento, llegaría un envío incompleto que generaría errores o entradas duplicadas.

✓ Con N8N como intermediario

N8N vigila el estado de planificación de cada transporte en SAP. Solo actúa cuando el transporte está completamente planificado: transportista asignado, fechas confirmadas, listo para ejecutar. En ese momento, y solo en ese momento, crea el envío en Tookane.

5Cada transportista tiene reglas específicas que SAP no conoce
⚠ Sin N8N — SAP directo a Tookane

SAP no sabe, por ejemplo, que Kuehne+Nagel necesita saber si el envío es para una empresa (B2B — entrega en un almacén o empresa) o para un particular (B2C — entrega en domicilio), ni que la referencia del pedido de Actiu debe ir en un campo concreto de su sistema, o que los números de cada bulto van en otro campo diferente. Sin ese conocimiento, los datos acaban en un cajón de sastre dentro de Tookane llamado EXTRAINFO — un campo genérico que el sistema de transportes no puede procesar de forma automática.

✓ Con N8N como intermediario

N8N tiene configuradas las reglas específicas de cada transportista. Para Kuehne+Nagel, por ejemplo, marca automáticamente todos los envíos como B2B, coloca la referencia del cliente de Actiu en el campo que K+N espera, y añade los números de cada bulto donde corresponde. Tookane recibe los datos exactamente como los necesita, sin tener que interpretar nada.

6SAP solo tiene los datos logísticos — falta el contexto operativo
⚠ Sin N8N — SAP directo a Tookane

SAP tiene los datos del transporte: qué se envía, a quién y cuándo. Pero no tiene el teléfono del contacto del transportista para llamar si hay un problema, ni si ese cliente tiene un acuerdo especial de nivel de servicio, ni qué tipo de incidencias ha habido antes con ese carrier en esa ruta.

✓ Con N8N como intermediario

Antes de crear el envío en Tookane, N8N añade automáticamente el contexto operativo: el contacto del transportista responsable de esa expedición, el perfil del cliente (urgencia, idioma de contacto) y el historial de incidencias con ese carrier. Tookane no solo recibe el envío — recibe el envío con toda la información para gestionarlo bien.
